# Scripts and Tests Explanation

## What Scripts Do

### `run_comparison_gatelevel.sh <test> <seed>`
Runs a test on the gate-level netlist and compares it to baseline Ibex.

**Steps:**
1. Calls baseline Ibex DV to generate and run a test with Spike co-sim
2. Copies the test binary to local directory
3. Runs gate-level simulation with the same binary
4. Compares instruction traces
5. Reports PASS/FAIL

**Example:**
```bash
./run_comparison_gatelevel.sh riscv_arithmetic_basic_test 1000
```

### `run_comparison_modified_rtl.sh <test> <seed>`
Same as above but for modified RTL instead of gate-level.

### What Tests Are Running

The tests are **random RISC-V programs** generated by the RISCV-DV framework.

**Test Generation:**
- RISCV-DV uses constrained random generation
- Creates assembly code exercising RISC-V instructions
- Each test has a unique seed for reproducibility
- Tests include setup code, random instructions, and a signature write at completion

**Test Types:**
- `riscv_arithmetic_basic_test`: ADD, SUB, AND, OR, XOR, shifts, etc.
- `riscv_rand_instr_test`: Random mix of all legal instructions
- `riscv_rand_interrupt_test`: Random interrupts during execution (for latency measurement)

**What's Being Verified:**
1. The modified/gate-level core executes the same instructions as baseline
2. Register writes produce the same values
3. Memory accesses go to the same addresses
4. Program flow matches (same PCs in same order)

**Interrupt Latency Measurement:**
For interrupt tests, the trace logs show:
- Cycle of interrupt assertion
- Cycle of first instruction in handler
- Latency = difference in cycles

```bash
./run_comparison_gatelevel.sh riscv_arithmetic_basic_test 1000

# Run single test (modified RTL) - currently broken
./run_comparison_modified_rtl.sh riscv_arithmetic_basic_test 1000

```
