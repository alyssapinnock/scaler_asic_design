## What Tests Are Running

The scripts run **random RISC-V programs** generated by RISCV-DV:

1. Baseline Ibex DV generates a random test (e.g., `riscv_arithmetic_basic_test`)
2. Test is compiled to a binary
3. Binary runs on baseline Ibex + Spike co-simulation
4. Spike verifies correctness (golden reference)
5. Same binary is loaded into your modified/gate-level core
6. Instruction traces are compared
7. If traces match → PASS, core is functionally correct
8. If traces differ → FAIL, shows first mismatch

**For interrupt latency:** Use `riscv_rand_interrupt_test` and measure cycles from interrupt assertion to handler entry in the trace logs.
